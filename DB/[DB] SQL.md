# Database

**개념**

- 체계적으로 정리된 데이터의 모음
- 수많은 정보를 빠르게 찾고 효율적으로 관리하기 위해, 데이터를 **구조적으로 저장**하는 역할
- DATABASE 요약
    - 테이블 : 데이터가 기록되는 곳
    - 테이블에는 행에서 고유하게 식별 가능한 기본 키라는 속성이 있으며, 외래 키를 사용하여 각 행에서 서로 다른 테이블 간의 관계를 만들 수 있음
    - 데이터는 기본 키 또는 외래 키를 통해 결합될 수 있는 여러 테이블에 걸쳐 구조화됨
    
    ### 🧱 테이블(Table)
    
    > 데이터가 실제로 저장(기록) 되는 기본 단위이다.
    > 
    > 
    > 각 테이블은 여러 **행(Row)** 과 **열(Column)** 로 구성되며,
    > 
    > 테이블의 구조는 스키마(Schema)에 의해 정의된다.
    > 
    
    ---
    
    ### 🔑 키(Key)
    
    - **기본 키(Primary Key)** :
        
        테이블 내에서 **각 행을 고유하게 식별**할 수 있는 속성이다.
        
        예) 회원 테이블의 `member_id`, 상품 테이블의 `product_id`
        
    - **외래 키(Foreign Key)** :
        
        다른 테이블의 기본 키를 참조하여 **테이블 간의 관계**를 형성한다.
        
        예) 주문 테이블의 `member_id` → 회원 테이블의 `member_id`
        
    
    ---
    
    ### 🧩 테이블 간의 관계 구조
    
    > 데이터는 하나의 테이블에만 존재하지 않고,
    > 
    > 
    > **기본 키 또는 외래 키를 통해 서로 연결될 수 있는 여러 테이블에 분산되어 구조화**된다.
    > 
    > 이렇게 함으로써 데이터 중복을 줄이고,
    > 
    > 필요한 경우 **여러 테이블을 결합(Join)** 하여 다양한 정보를 조회할 수 있다.
    > 

**역할**

- 데이터를 **저장(구조적 저장)**하고 조작(CRUD)
    - **C**reate
    - **R**ead
    - **U**pdata
    - **D**elete
    

## 데이터

**개념**

- 저장이나 처리를 위해 변환된 정보
- 표현한 정보들을 컴퓨터가 다룰 수 있도록 구조화한 것
    - 분석, 계산, 저장을 위해 정보를 구체적인 형태로 정리한 것
- 데이터 사용량, 데이터 센터의 성장 등의 이유로 데이터 저장 및 관리하여 활용할 수 있는 기술 중요

**저장 방식**

1. 파일(File)
    - 장점 : 어디에서나 쉽게 사용 가능
    - 단점 : 데이터를 구조적으로 관리하기 어려움
2. 스프레드 시트(Spreadsheet)
    - 테이블의 열과 행을 사용해 데이터를 구조적으로 관리 가능
    - 한계점
        - 데이터의 크기 : 일반적으로 약 100만 행까지만 저장 가능
        - 보안적 측면 : 단순히 파일이나 링크 소유 여부에 따른 단순한 접근 권한 기능 제공
        - 정확성 측면
            - 데이터가 여러 시트에 분산된 경우 변경 시 누락이 생기거나 추가 문제가 발생할 수 있음
            - 하나의 누락이 생기면 그 것에서 파생되는 문제점들이 나타날 것

## 관계형 데이터베이스

**Relational Database**

**개념**

- 데이터 간에 **관계**가 있는 데이터 항목들의 모음
    - 관계 : 여러 테이블 간의 (논리적) 연결
        - 데이터를 각각의 테이블에 나눠 저장하되, 공통된 키 값을 통해 서로 연결하여 필요할 때 함께 조회하거나 조작 가능하도록 함
        - 데이터를 다양한 형식으로 조회 가능하게 됨
    - 고유한 번호를 부여하고, 해당 번호를 기록에 남겨 개별 정보와 기록을 정확히 연결 가능
        - **기본 키 (Primary Key)** : 각 데이터에 부여한 고유 식별 값
        - **외래 키 (Foreign Key)** : 다른 기본 키를 참조하여 저장한 고유한 식별 값
- 테이블, 행, 열의 정보를 구조화하는 방식
    - 서로 관련된 데이터 포인터를 저장하고 이에 대한 액세스 제공
- 데이터베이스 : 데이터를 **저장(구조적 저장)**하고 조작(CRUD)
    - 관계형 데이터베이스의 구조란,
    데이터를 테이블 형태로 저장하고, 각 테이블 간의 관계를 통해 데이터를 연결하는 것
    - 단순히 관계가 있는 것만이 아닌, 데이터가 일정한 형식(스키마)에 따라 정리되어 있는 구조 전체
    

**관련 키워드**

1. Table
    - Relation
    - 데이터를 기록하는 곳
2. Field
    - Column, Attribute
    - 각 필드에는 고유한 데이터 형식(타입)이 지정됨
3. Record
    - Row, Tuple
    - 각 레코드에는 구체적인 데이터 값이 저장됨
4. Database
    - Schema
    - 테이블의 집합
5. Primary Key
    - 기본 키 (PK)
    - 각 레코드의 고유한 값
    - 관계형 데이터 베이스에서 **레코드의 식별자**로 활용
6. **Foreign Key**
    - 외래 키 (FK)
    - 테이블의 필드 중 다른 테이블의 레코드를 식별할 수 있는 키
    - 다른 테이블의 기본 키를 참조
    - 각 레코드에서 서로 다른 테이블 간의 **관계를 만드는 데** 사용

## **DBMS**

**Database Management System**

**개념**

- 데이터베이스를 관리하는 소프트웨어 프로그램
    - 데이터를 일정한 규칙으로 저장하고, 필요할 때 쉽게 꺼내거나 수정할 수 있도록 돕는 도구 역할
- 데이터의 저장, 관리를 용이하게 하는 시스템
- 데이터 베이스와 사용자 간의 인터페이스 역할
- 사용자가 데이터 구성, 업데이트, 모니터링, 백업, 복구 등을 할 수 있도록 도움

## RDBMS

**Relational Database Management System**

**개념**

- 관계형 데이터베이스 관리 소프트웨어 프로그램
- 데이터를 여러 테이블에 나누어 저장하되, 공통된 키를 통해 서로 관계를 맺고 함께 사용할 수 있도록 해주는 시스템
- 프로젝트의 목적과 환경에 따라 선택
    - 모두 SQL을 기반으로 동작하지만, 세부 기능이나 사용 방식은 조금씩 다름

**종류**

- SQLite
    - 경량의 오픈 소스 데이터베이스 관리 시스템
    - 설치 없이 가볍게 실행 가능하여 모바일 앱이나 소규모 프로그램에 적합함
    - 컴퓨터나 모바일 기기에 내장되어 간단하고 효율적인 데이터 저장 및 관리 제공
- MySQL
- PostgreSQL
- Oracle Database 등

# SQL

**Structure Query Language**

**개념**

- 테이블의 형태로 구조화된 관계형 데이터베이스에 요청을 질의(요청)
    - 구조적 Query
    - 데이터베이스에서 사서인 DBMS에게 원하는 데이터를 찾아달라고 요청하는 표준화된 요청서 양식
    - 사람이 사용하는 모호한 말이 아닌 **명확하게 약속된 형식**에 따라 데이터베이스에 정확한 요청을 전달하는 언어

## SQL Statements

**개념**

- SQL을 구성하는 가장 기본적인 코드 블록
- 작성 방식
    - SQL 키워드
        - 대소문자 구분하지 않음
        - 대문자로 작성하는 것을 권장 (명시적 구분)
    - SQL Statements 끝에는 세미콜론(;) 필요
        - 각 SQL Statements를 구분하는 방법
        - 명령어의 마침표의 역할

**유형**

- 수행 목적에 따른 구분
    - **DDL (**Data Definition Language)
        - 데이터 정의
        - 데이터의 기본 구조 및 형식 변경
        - 키워드
            - CREATE
            - DROP
            - ALTER
    - **DQL (**Data Query Language)
        - 데이터 검색
        - 키워드
            - SELECT
    - **DML (**Data Manipulation Language)
        - 데이터 조작
            - 추가, 수정, 삭제
        - 키워드
            - INSERT
            - UPDATE
            - DELETE
    - **DCL (**Data Control Language)
        - 데이터 제어
        - 데이터 및 작업에 대한 사용자 권한 제어
        - 키워드
            - COMMIT
            - ROLLBACK
            - GRANT
            - REVOKE

## NULL

개념

- SQL에서 NULL은 실제 값이 아니라 “값이 없음” 또는 “알 수 없음”을 의미
- 일반적인 등호(’=’) 연산자로 NULL을 비교하면 의도한 대로 작동하지 않음

## SQL

- 논리 연산에 대해 3가지 값을 사용
    - TRUE
    - FALSE
    - UNKNOWN
        - NULL = NULL : TRUE가 아닌 UNKNOWN
        - 실제 어떤 값을 가지고 있는 것이 아니기 때문
