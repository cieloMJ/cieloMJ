# 1. HTML의 기본 구성 요소

## 1) 태그 
- 다양한 구성 요소를 정의하는 역할
- HTML 문법을 이루는 가장 작은 단위
- 기본 형식 : 홑화살괄호(<>) 사이에 태그명 작성
  - <태그명>

## 2) 속성
- 태그에 어떠한 의미나 기능을 보충하는 역할
  - 태그 없이 사용 불가
- 일종의 옵션응로 사용 여부, 여러 개 사용 등 모두 가능
- 기본 형식 : 속성명과 속성값으로 구성
  - <태그명 속성명="속성값">
    - 속성명 : 따옴표 없이 작성
    - 속성 값
      - 큰 따옴표("") 안에 작성
      - 속성 값이 여러 개인 경우, 하나의 큰 따옴표 안에 쉼표(,)로 구분해 값 나열
- 예시
  <html lang="ko">
  - lang : 속성명
  - ko : 속성값
  - html 태그만 사용했다면 HTML 문서의 시작이라는 의미만 있음
    lang 속성 때문에 태그의 원래 의미에 속성의 의미가 더해져 주 언어가 한글로 된 HTML 문서의 시작이라는 구체적인 의미 가지게 됨

## 3) 문법
- HTML : 태그와 속성으로 문법 구성
- 문법
  - 콘텐츠(content)가 있는 문법
    - 콘텐츠 앞뒤를 태그로 감쌈
      - 시작 태그(Open Tag) : 앞에 넣는 태그
      - 종료 태그(close Tag) : 뒤에 넣는 태그
        - 태그명 앞에 슬래시(/)가 있음
    - 요소(element) : 시작 태그 + 콘텐츠 + 종료 태그
      - 예시 : <title>My First Web Page</title>
  - 콘텐츠가 없는 문법
    - 시작 태그만 사용
      - 시작 태그가 곧 요소
      - 예시 : <br>
    - 내용이 비어 있다는 의미로 빈 태그(empty tag)라고도 함

## 4) 주석 
-  코드에 어떠한 메모나 설명을 남기고 싶을 때 사용
-  웹 브라우저(실행 결과)에서는 표시되지 않음
  - 단, 소스 보기로 볼 수 있으므로 보안상 중요한 내용을 넣으면 안됨
-  <!-- 주석 내용 -->


# 2. HTML 기본 구조
- 일정한 구조 안에서 작성됨 (기본 구조)

## 1) DTD 
- 문서형 정의 (DTD; Document Type Definition)
- 웹 브라우저가 처리할 HTML 문서가 어떤 형식을 따라야 하는지 알려주는 것
  - HTML 문서를 작성할 때 가장 처음에 넣어야 함
  - <!DOCTYPE html>

## 2) html 태그
- HTML 문서의 시작과 끝 의미
- 모든 태그는 html 태그 안에 작성되어야 함

## 3) head 태그
- HTML 문서의 메타데이터(metadata)를 정의하는 영역
  - 메타데이터란?
    HTML 문서에 대한 정보(data)로, 웹 브라우저에는 직접 노출되지 않음
- meta, title, link, style, script 등의 태그를 사용하여 HTML 문서의 여러 정보를 정의

### meta 태그
- 메타데이터를 정의하는 데 사용
- <meta charset="UTF-8">
  : HTML 문서에서 허용하는 문자 집합(charset)을 정의하는 메타데이터 태그
- <meta http-equiv="X-UA-Compatible" content="IE=edge">
  : 인터넷 익스플로어(IE)의 렌더링 엔진을 강제로 최신 렌더링 엔진으로 지정하는 메타데이터 
  (구 버전의 렌더링 엔진을 사용하는 프로젝트가 아니라면 해당 태그를 정의하는 것이 좋음)
- <meta name="viewport" content="width=device-width, initial-scale=1.0">
  : 기기의 화면 너비에 맞추기 위해 사용하는 메타데이터 태그
  (뷰포트: 웹페이지에 접속했을 때 사용자에게 보이는 화면 영역)

### title 태그
- HTML 문서의 제목을 지정하는 데 사용
  - <title>My First Web Page</title>
- 반드시 1개의 title 태그를 사용해 HTML 문서의 제목을 지정해야 함
  - 이때 제목은 HTML 문서마다 중복되지 않도록 주의해야 함
  - 검색 엔진 사이트에서 HTML 문서를 찾을 때 title 태그에 작성된 제목으로 찾기 때문
    (한 웹 사이트에서 제목이 중복된 문서가 여러 개 발견된다면 검색 엔진은 해당 웹 사이트가 신뢰성이 떨어진다고 판단하여 검색 엔진 노출 시에 불이익을 줌)

## 4) body 태그
- 웹 브라우저에 노출되는 내용을 작성하는 영역


# 3. HTML 특성
## 1) 블록 요소와 인라인 요소
### a. 블록 요소
- body 태그에서 사용하는 태그 중에서 웹 브라우저의 공간 유무와 상관없이 hn 태그나 p 태그처럼 사용할 때마다 줄바꿈 되는 태그로 작성한 코드

### b. 인라인 요소
- a 태그나 span 태그처럼 공간이 부족할 때만 줄바꿈되는 태그로 작성한 코드

## 2. 부모, 자식, 형제 관계
- HTML은 태그를 사용할 때 부모, 자식, 형제라고 하는 관걔가 성립됨
- 예시
  - head 태그 & body 태그 : html 태그의 자식
  - html 태그 : head 태그 & body 태그의 부모
  - head 태그 - body 태그 : 부모가 같으므로 형제 관계
  ( meta, title 태그도 마찬가지 )
- 태그의 사용 위치에 따라 부모, 자식, 형제 관계를 맺음
- 조상과 자손(후손)의 관계도 존재
  - 조상 : 부모의 부모 태그
  - 자손(후손) : 자식의 자식 태그

## 3. 줄 바꿈과 들여쓰기
- HTML 문서에 태그를 작성할 때는 되도록 줄 바꿈과 들여쓰기를 하는 것이 좋음
  - 문법상 강제된 규칙 아님
  - 암묵적으로 치키는 규칙
- 코드의 가독성을 높일 수 있는 가장 쉬운 방법
  - 좋은 HTML 문서는 코드의 가독성도 좋아야 함
  - 읽기 좋은 코드는 나중에 다시 코드를 확인하거나 수정해야 할 때 코드 전체 내용을 금방 파악 가능
